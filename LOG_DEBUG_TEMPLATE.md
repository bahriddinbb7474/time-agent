# Log Debug Template — Time-Agent

Этот шаблон используется для диагностики ошибок через анализ логов.

Основной принцип:

**Сначала анализ логов → потом исправление кода.**

Codex сначала должен определить причину ошибки, а не сразу менять код.

---

# Сценарий

Что делал пользователь.

Пример:

- отправил команду /add
- запустил scheduler
- синхронизировал Google Calendar
- запустил Docker контейнер

---

# Ожидаемый результат

Что должно было произойти.

Пример:

- задача должна была создаться
- бот должен был ответить
- событие должно было синхронизироваться
- сервис должен был запуститься

---

# Фактический результат

Что произошло на самом деле.

Пример:

- бот не ответил
- команда не сработала
- произошла ошибка
- сервис упал

---

# Лог / Traceback

Вставить лог ошибки или traceback.

Пример:
Traceback (most recent call last):
File "handlers/task.py", line 52
KeyError: 'deadline'


---

# Задача для Codex

Codex должен:

1. Проанализировать лог или traceback
2. Определить точку сбоя
3. Найти связанные файлы
4. Сформулировать вероятную причину ошибки
5. Указать минимальную область исправления

---

# Ограничения

На этапе анализа:

- код **не изменять**
- новые файлы **не создавать**
- только анализ

---

# Формат ответа Codex

Codex должен ответить:

1. Симптом ошибки
2. Вероятная причина
3. Связанные файлы
4. Минимальная область изменения
5. Что проверить вручную

После ответа Codex **останавливается и ждёт следующую задачу**.

---

# Следующий шаг

После анализа:

Конструктор должен решить:

1. дать Codex задачу на минимальное исправление
или
2. вернуть вопрос Инженеру если проблема в постановке шага.
