# Fast Bug Scan — Time-Agent

Используется для быстрого поиска потенциальных ошибок во всём проекте.

Основной принцип:

Fast Bug Scan выполняет **только анализ**.
Код не изменяется.

---

# Когда запускать

- после крупных изменений
- перед релизом
- при странном поведении системы
- периодически во время разработки

---

# Задача для Codex

Codex должен:

1. Проанализировать Python файлы проекта
2. Найти потенциальные проблемы:

- возможные исключения
- ошибки async / await
- ошибки scheduler
- неправильную работу с БД
- неинициализированные переменные
- возможные NoneType ошибки
- неправильные импорты
- race conditions
- небезопасную обработку API
- возможные ошибки Telegram handlers

---

# Ограничения

- код не изменять
- файлы не переписывать
- только анализ

---

# Формат ответа Codex

1. Тип проблемы
2. Файл
3. Строка
4. Причина риска
5. Критичность (low / medium / high)

После ответа Codex должен остановиться.

---

# Следующий шаг

После анализа:

Конструктор выбирает **одну проблему** и готовит отдельную задачу для исправления.
