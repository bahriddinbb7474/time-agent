# Codex Task Protocol — Time-Agent

Для задач, связанных с изменением логики, используется 3-Step Task Protocol.

---

## Step 1 — ANALYZE

Цель: определить точку изменения.

Codex должен:

- найти связанные файлы
- определить точку изменения
- оценить риски

Код не изменяется.

---

## Step 2 — IMPLEMENT

Цель: внести минимальное изменение.

Правила:

- minimal change
- diff-first
- не менять архитектуру
- менять только указанную область

---

## Step 3 — VERIFY

Цель: проверить последствия изменения.

Codex должен:

- проверить связанные модули
- оценить side effects
- указать риски

Код не изменяется.

---

## Основное правило

Не объединять ANALYZE, IMPLEMENT и VERIFY в одну задачу.

Каждый шаг выполняется отдельно.
