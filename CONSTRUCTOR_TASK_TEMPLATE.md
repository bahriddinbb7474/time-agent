# Constructor → Codex Task Template

Используется для передачи задач от Конструктора к Codex.

---

## Контекст
Краткое описание модуля или проблемы.

## Текущий шаг
Один конкретный шаг реализации.

## Цель
Какой результат должен быть достигнут.

## Ограничения
- не менять архитектуру проекта
- менять минимальное количество файлов
- не выполнять следующий шаг автоматически
- не делать рефакторинг без запроса

## Что нужно сделать
1. Найти нужные файлы
2. Внести минимальные изменения
3. Показать результат изменений
4. Написать как проверить

## Формат ответа Codex
1. Что сделано
2. Какие файлы изменены
3. Как проверить
4. Возможные риски

После выполнения задачи Codex должен остановиться и ждать следующую команду.

- не изменять другие файлы
- не выполнять дополнительные действия
